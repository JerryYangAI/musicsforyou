# 端口冲突解决方案

## 🔍 问题诊断

端口 5000 被 macOS 的 **AirPlay 接收器**服务占用，导致服务器无法启动。

## ✅ 解决方案

### 方案1: 更改服务器端口（推荐）

我已经将端口改为 **3000**，现在可以正常启动：

```bash
npm run dev
```

服务器将在 `http://localhost:3000` 启动。

### 方案2: 禁用AirPlay接收器（如果想继续使用5000端口）

1. 打开 **系统设置** (System Settings)
2. 进入 **通用** (General) → **隔空投放与接力** (AirDrop & Handoff)
3. 关闭 **隔空播放接收器** (AirPlay Receiver)

然后可以改回使用端口 5000。

### 方案3: 使用其他端口

如果3000也被占用，可以改为：
- 8000
- 8080
- 3001
- 5001

在 `.env` 文件中修改 `PORT` 值即可。

**当前配置**: 端口已改为 3000
